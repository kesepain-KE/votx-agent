# kesepain-Agent 开发文档

> 最后更新: 2026-05-01

## 一、项目概述

多用户 AI Agent 框架。以 OpenAI 兼容接口调用 LLM，支持角色扮演、工具调用、对话持久化，内置安全机制（权限/沙箱/限流/日志）。

**技术栈**: Python 3.10+ / `openai` SDK / `requests` / DeepSeek API（可替换任意 OpenAI 兼容接口）

**当前状态**: 生产可用的闭环 — 用户选择 → 加载人设+AGENT.md → 对话 → 工具调用 → Token 统计 → 历史归档。14 个工具，8 个 Skill。

## 三、运行流程

### 3.1 整体调用链

```
start.py
  │
  ├─ ① 扫描 users/ 目录，列出可用用户
  ├─ ② 用户输入编号选择
  ├─ ③ 设置环境变量 KESEPAIN_USER_DIR
  └─ ④ subprocess.run() 调用 main.py
       │
       ▼
main.py
  │
  ├─ 1. 从环境变量 KESEPAIN_USER_DIR 获取用户目录
  ├─ 2. 加载 config/config_core.json（全局配置）+ .env
  ├─ 3. 加载 users/<name>/config.json（用户配置）
  ├─ 4. 组装 system prompt:
  │     ├─ self_soul.md（角色人设）
  │     ├─ config/soul.md（全局补充，可选）
  │     └─ AGENT.md（项目能力指南，让 LLM 了解自身工具和架构）
  │
  ├─ 5. 初始化:
  │     ├─ ChatManager → load_history()
  │     ├─ DeepSeekProvider（校验 API Key，加载 .env）
  │     ├─ register_all() → 按 SKILL.md 发现 + 加载 tool.py
  │     └─ ToolRunner(core_config, user_config)
  │
  └─ 6. 对话循环 → 详见 3.2
```

### 3.2 对话循环详解

```
while True:
    user_input = input("您: ")

    # 斜杠命令分发
    if user_input.startswith("/"):
        result = _dispatch(user_input)
        /exit, /q       → True  (退出循环)
        /clear          → False (清除历史+归档，继续)
        /history,/stats → False (显示统计，继续)
        /archive        → False (手动归档，继续)
        /help           → False (显示帮助，继续)
        未知命令        → None  (交给LLM处理)

    chat.add_user_message(user_input)
    tool_runner.reset_count()
    round_usages = []  # 收集本轮流 token 用量

    # 内层循环: 处理 tool call 往返（上限 MAX_TOOL_ROUNDS=20）
    while tool_round < MAX_TOOL_ROUNDS:
        messages = chat.build_messages()         # system + history
        response = provider.chat(messages, tools)

        # 记录本轮 token 用量
        if provider.last_usage:
            round_usages.append(provider.last_usage)

        if tool_runner.has_tool_calls(response):
            chat.add_tool_call_message(...)       # 助手消息 (tool_calls)
            results = tool_runner.execute(...)    # 权限→限流→执行→日志
            chat.add_tool_results(results)
            # 工具 ERROR 直接交 LLM，让它自行判断
            print(f"[工具调用: N 个] [入 xxx (命中 n) | 出 xxx | 共 xxx]")
        else:
            print(f"助手: {response.content}")
            chat.add_assistant_message(...)
            break

    # 本轮对话结束，输出累计 token
    print(f"[Token: 输入 xxx (缓存命中 n) | 输出 xxx | 总计 xxx]")

    chat.save_history()   # 落盘历史
    chat.save_log(...)    # 落盘完整日志（含 system prompt）
```

### 3.3 斜杠命令

| 命令 | 功能 |
|------|------|
| `/exit`, `/quit`, `/q` | 退出（自动保存历史） |
| `/clear` | 清除当前对话历史（自动归档备份后清空） |
| `/history`, `/stats` | 查看历史状态（消息数/文件大小/归档数） |
| `/archive` | 手动归档当前历史（不清空） |
| `/help` | 显示命令帮助 |

### 3.4 Token 显示

每次 LLM API 调用后自动统计并显示：

- **每轮 (per-round)**: 工具调用行后显示 `[入 xxx (命中 n) | 出 xxx | 共 xxx]`
- **累计 (cumulative)**: 本轮对话结束后显示总计 `[Token: 输入 xxx (缓存命中 n) | 输出 xxx | 总计 xxx]`
- **缓存命中**: 括号内 `(命中 n)` 表示 prompt 缓存命中的 token 数——命中越高越省钱

## 四、核心模块 API

### 4.1 provider/openai_api.py

```python
class DeepSeekProvider:
    def __init__(self, user_config: dict)
        # 从 user_config["provider"] 读取:
        #   api_key — 优先 config，fallback 环境变量 DEEPSEEK_API_KEY
        #             为空时抛出 ValueError（含设置指引）
        #   model   — 默认 "deepseek-v4-flash"
        #   think   — 默认 False → thinking: disabled
        #   stream  — 默认 False → 非流式
        #   timeout — 默认 120s

        # .env 加载顺序:
        #   1. <项目根>/.env
        #   2. <CWD>/.env
        #   不覆盖已存在的环境变量

    last_usage: dict | None  # 最近一次 API 调用的 token 用量
        # {"prompt_tokens": N, "completion_tokens": N,
        #  "total_tokens": N, "cached_tokens": N}

    def chat(messages, tools=None) -> ChatCompletionMessage
        # think=True → reasoning_effort="high"
        # think=False → extra_body={"thinking":{"type":"disabled"}}
        # stream=True → 收集 chunk 拼接为完整 Message + usage
        # 自动重试: 超时/连接错误最多重试 2 次
```

### 4.2 run/chat.py

```python
class ChatManager:
    # 初始化: user_dir, core_config, user_config
    # 配置: history_max(100) / zip_history(true)

    # --- 消息管理 ---
    set_system_prompt(prompt)          # 设置角色人设
    build_messages() -> list[dict]      # system + self.messages

    add_user_message(content)           # {"role":"user",...}
    add_assistant_message(content)      # {"role":"assistant",...}
    add_tool_call_message(tool_calls)   # SDK 对象 → dict 后追加
    add_tool_results(results)           # [{"role":"tool",...}]

    # --- 持久化 ---
    load_history()                      # JSON → self.messages（损坏自动修复）
    save_history()                      # self.messages → JSON（不含system）
    save_log(full_messages)             # 完整消息链（含system）→ JSON

    # --- 历史管理命令 ---
    clear_history() -> str              # 先归档再清空，删除历史文件
    archive_now() -> str                # 手动归档当前全部历史（不清空）
    history_stats() -> str              # 返回状态摘要（消息数/大小/归档数）

    # --- 内部 ---
    _trim_if_needed()                   # >= max_history 时裁剪
    _archive(old_messages)              # zip_history 时归档到 archive/
```

**消息存储格式**（OpenAI 标准 JSON）:

```json
[
  {"role": "system", "content": "角色人设..."},
  {"role": "user", "content": "你好"},
  {"role": "assistant", "content": "你好呀！"},
  {"role": "assistant", "content": null,
   "tool_calls": [{"id":"call_xxx","type":"function",
    "function":{"name":"get_time","arguments":"{}"}}]},
  {"role": "tool", "tool_call_id": "call_xxx", "content": "OK: ..."}
]
```

## 五、安全机制

### 5.1 错误标准化

所有异常统一 `ERROR: xxx` 前缀，LLM 可据此判断调用是否成功：

```python
from skills._common import err
err("文件不存在: /path")  # → "ERROR: 文件不存在: /path"
```

成功返回不加前缀，或以 `OK:` 开头。

### 5.2 路径沙箱

file 工具只允许访问：
- 用户目录（环境变量 `KESEPAIN_USER_DIR`）
- 项目根目录

越权路径 `safe_path()` 返回 `None`，调用方返回 `ERROR: 路径越权或无效`。write_file 越权时自动回退到用户目录下的 basename。

### 5.3 权限系统

| 层级 | 字段 | 优先级 |
|------|------|--------|
| 全局 | `config_core.json → tool.enabled` | 低 |
| 用户 | `config.json → tool.enabled` | 中（覆盖全局） |
| 用户 | `config.json → tool.deny` | **最高**（黑名单） |

```json
// 示例：禁止删除文件，禁用 HTTP
{
  "tool": {
    "enabled": {"http_get": false},
    "deny": ["delete_file"]
  }
}
```

### 5.4 工具限流

| 限制 | 字段 | 默认 |
|------|------|------|
| 单轮总调用 | `tool.tool_max` | 100 |
| 单工具单轮 | `tool.tool_max_per_type` | 50 |
| 单轮往返上限 | `main.MAX_TOOL_ROUNDS` | 20 |

### 5.5 工具日志

`ToolRunner.execute()` 自动记录到 `history/log/tool_log.json`（JSON Lines）：

```json
{"ts":"2026-05-01T09:10:44+00:00","tool":"get_time","args":{},"result":"OK: ...","success":true,"elapsed":0.001}
{"ts":"2026-05-01T09:10:45+00:00","tool":"read_file","args":{"path":"/etc/passwd"},"result":"ERROR: 路径越权","success":false,"elapsed":0.0}
```

### 5.6 Shell 安全

`run_command` 不限制命令（shell=False + shlex.split）：
- 参数级黑名单拦截危险模式（rm -rf /, mkfs, shutdown, chmod 777 / 等）
- 120 秒超时，适应下载等长任务
- 强制 UTF-8 编码 + errors=replace（避免 Windows GBK 崩溃）

### 5.7 HTTP SSRF 防护

`http_get`/`http_post` 内置多层防护：
- 仅允许 http/https 协议
- 拦截所有内网地址段（10.x, 172.16.x, 192.168.x, 127.x 等）
- DNS 解析后二次校验 IP（防 DNS rebinding）
- 响应 8000 字符截断上限

### 5.8 输出截断

file / http / shell 返回内容超过 8000 字符自动截断，标注 `...[截断: 原始 N 字符]`。

## 七、Skill 标准骨架

### 7.1 规范

遵循 **agentskills.io** 标准，每个 Skill 是一个独立目录：

```
skill-name/
├── SKILL.md      ← 身份证：YAML frontmatter (name + description) + Markdown 使用指引
├── tool.py       ← 执行体：包含 register()，注册 OpenAI tool_call schema + handler
├── scripts/      ← 可选：内部脚本（不被自动发现）
├── references/   ← 可选：参考文档、平台映射等
└── assets/       ← 可选：模板/静态资源
```

### 7.2 发现规则

`skills/__init__.py → register_all()`:
1. 递归扫描 `skills/` 下所有 `SKILL.md`
2. 对每个 SKILL.md，加载同目录的 `tool.py`
3. 调用 `mod.register()` 注册 tool_call
4. `_` 开头的目录/文件视为内部模块，不当作 Skill
5. `tool.py` 以外的 `.py`（如 scripts/ 下）不会被扫描

### 7.3 添加新 Skill

创建目录结构即可，无需修改任何注册代码：

```bash
skills/my-skill/
├── SKILL.md       # name + description + 使用指引
└── tool.py        # 包含 register()
```

```python
# skills/my-skill/tool.py
from run.tool import register_tool
from skills._common import err, truncate

def my_func(param: str) -> str:
    try:
        return f"OK: {param}"
    except Exception as e:
        return err(f"失败: {e}")

SCHEMA = {
    "type": "function",
    "function": {
        "name": "my_func",
        "description": "功能说明",
        "parameters": {
            "type": "object",
            "properties": {
                "param": {"type": "string", "description": "参数说明"}
            },
            "required": ["param"]
        }
    }
}

def register():
    register_tool(SCHEMA, my_func)
```

```markdown
# skills/my-skill/SKILL.md
---
name: my-skill
description: 当用户需要 xxx 时使用此技能
---

# 我的技能

具体的使用指引...
```

## 八、配置说明

### 8.1 全局配置 config/config_core.json

```json
{
    "history": {
        "history_max": 100,        // 最大消息条数，超限触发裁剪
        "zip_history": true        // 裁剪时归档旧消息到 history/archive/
    },
    "tool": {
        "tool_max": 100,           // 单轮最大工具调用次数
        "tool_max_per_type": 50,   // 单个工具单轮最大调用次数
        "tool_log_max": 1000,      // 工具日志条数上限（预留）
        "enabled": {}              // 全局工具开关，例 {"delete_file": false}
    }
}
```

### 8.2 用户配置 users/<name>/config.json

```json
{
    "provider": {
        "name": "deepseek",
        "model": "deepseek-v4-flash",
        "api_key": "",
        "think": false,
        "stream": false,
        "timeout": 120
    },
    "history": {
        "data": "chat_data.json",
        "log": "chat_log.json"
    },
    "tool": {
        "enabled": {},
        "deny": []
    }
}
```

### 8.3 角色灵魂 self_soul.md

纯文本 Markdown，作为 system prompt 发送给 LLM。包含角色名称、能力范围、回复格式、情感风格等。

### 8.4 API Key 设置方式

按优先级：
1. `users/<name>/config.json → provider.api_key`（用户级）
2. 环境变量 `DEEPSEEK_API_KEY`
3. 项目根 `.env` 文件（自动加载，不覆盖已有环境变量）

**额外密钥**：`UAPI_API_KEY`（热榜查询）、`HTTP_TIMEOUT`（HTTP 超时）等也可在 `.env` 中配置。

### 8.5 权限优先级

```
tool.deny（用户黑名单）  >  tool.enabled（用户级）  >  tool.enabled（全局）
```

### 8.6 禁用工具

```json
// 全局关闭: config/config_core.json
{"tool": {"enabled": {"delete_file": false}}}

// 用户黑名单: users/<name>/config.json
{"tool": {"deny": ["delete_file", "run_command"]}}
```
