## 文档分层

四层文档，各司其职：

| 文件 | 面向 | 内容 |
|---|---|---|
| `README.md` | 用户 | 快速启动、功能概览 |
| `config/soul.md` | 运行时 AI | 全局人格基座：诚实、安全、隐私 |
| `AGENTS.md` | 编码 Agent | 执行规则、路径、坑、安全边界 |
| `开发文档/` | 维护者 | 完整技术细节，只记已实现事实 |

用户人设 `users/<name>/self_soul.md` 在全局基座之上叠加，定义角色风格（如猫猫小助手）。

## 核心链路

```
入口: start.py (CLI) / start_web.py (Web)
  → 选择用户 → 加载配置
  → 初始化 Provider / ToolRunner / ChatManager
  → build_system_prompt()
      1. self_soul.md          用户人设
      2. config/soul.md        全局人格基座
      3. AGENTS.md             编码手册
      4. Skill 摘要            工具 + 指令
      5. agent_memory.db       持久记忆 (≤20)
      6. .learnings/           学习记录
  → run_chat_turn
      → tool_calls 循环 (reasoning_content 回传, max 20 轮)
      → 保存历史 / 日志 / 归档

Web 额外: Flask SSE 事件流 → Vue 3 前端渲染
```

## 启动命令

```bash
python start.py                  # CLI
python start.py --web            # Web (13579)
python start_web.py              # Web (1478, 冲突自动轮询)
python start_web.py --port=8080  # 指定端口
```

## Web 命令

| 命令 | 说明 |
|---|---|
| `/clear` | 清空当前对话及工具日志 |
| `/history` | 会话统计 |
| `/retry` | 移除上轮 AI 回复，重新生成 |
| `/help` | 帮助 |

`/exit` 仅 CLI 可用。`/archive` `/summarize` 通过「保存」按钮触发。

## 文档同步规则

以下变更必须同步 README、AGENTS.md 和开发文档：

- 启动方式、端口、依赖
- Web 路由
- Skill 数量、工具数量
- system prompt 组合逻辑（含 soul.md / self_soul.md）
- 工具调用事件类型
- 历史保存 / 归档 / 断链修复
- 安全策略
- 用户可见 UI 行为
