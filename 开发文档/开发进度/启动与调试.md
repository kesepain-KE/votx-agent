# 启动与调试

最后更新：2026-05-04

## 启动

```bash
# CLI
python start.py

# Web (端口 13579)
python start.py --web
python start.py --web --port=8080

# Web 专用入口 (端口 1478, 冲突自动轮询)
python start_web.py
python start_web.py --port=8080
```

## 环境安装

```bash
python setup.py          # 检查环境 + 安装依赖 + 配置 .env
python setup.py --check  # 仅检查，不改动
python setup.py --skip-env  # 跳过 .env 配置
```

## 依赖

```bash
pip install -r requirements.txt
```

核心: openai, requests, yt-dlp, tavily-python, python-docx, pyyaml, flask, pytest

可选: pdfplumber, PyPDF2, gallery-dl, aria2

## MSYS2 / MINGW64 特殊处理

`python` 可能是 `D:/MSYS32/mingw64/bin/python.exe`。遇到 `No module named pip` 或 `externally-managed-environment` 时，不要强行 pip install。

Flask 用 pacman: `pacman -S --needed mingw-w64-x86_64-python-flask`

## 环境检查

```bash
python -c "import sys; print(sys.executable); print(sys.version)"
```

```bash
python - <<'PY'
import importlib.util
mods = ['openai', 'requests', 'yt_dlp', 'tavily', 'docx', 'yaml', 'flask', 'pytest']
for mod in mods:
    print(mod, 'ok' if importlib.util.find_spec(mod) else 'missing')
PY
```

## 缓存清理

```bash
python - <<'PY'
from pathlib import Path
import shutil
for p in Path('.').rglob('__pycache__'):
    if p.is_dir(): shutil.rmtree(p)
for pat in ('*.pyc', '*.pyo'):
    for p in Path('.').rglob(pat):
        if p.is_file(): p.unlink()
PY
```

## 常见问题

### tool_calls 断链

`run/chat.py` 含两阶段修复。仍报错时检查 `users/<name>/history/chat/*.json`，先备份再修复。

### tool_log.json 解析错误

是 JSONL (每行一个 JSON)，不是单个 JSON 文档。

### PDF 读取

用 PyPDF2 或 pdfplumber，不要用 `read_file` 或 python-docx。

### WSL vs Windows 路径

WSL 路径 `/mnt/e/...` 不能直接传给 Windows 程序，用 `cmd.exe /c` 包裹。
