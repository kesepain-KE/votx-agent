# 运行流程

## 整体调用链

```
start.py
  │
  ├─ ① 扫描 users/ 目录，列出可用用户
  ├─ ② 用户输入编号选择
  ├─ ③ 设置环境变量 KESEPAIN_USER_DIR
  └─ ④ subprocess.run() 调用 main.py
       │
       ▼
main.py
  │
  ├─ 1. 从环境变量 KESEPAIN_USER_DIR 获取用户目录
  ├─ 2. 加载 config/config_core.json + project/.env
  ├─ 3. 加载 users/<name>/config.json
  ├─ 4. 组装 system prompt:
  │     ├─ self_soul.md（角色人设）
  │     ├─ config/soul.md（全局补充，可选）
  │     └─ AGENT.md（项目能力指南）
  │
  ├─ 5. 初始化:
  │     ├─ ChatManager → load_history()
  │     ├─ DeepSeekProvider（校验API Key，加载 .env）
  │     ├─ register_all() → 按 SKILL.md 发现 → 加载 tool.py → 注册
  │     └─ ToolRunner(core_config, user_config)
  │
  └─ 6. 对话循环
```

## 对话循环详解

```
while True:
    user_input = input("您: ")

    # === 斜杠命令分发 ===
    if user_input.startswith("/"):
        result = _dispatch(user_input)
        /exit, /quit, /q  → True   (退出循环)
        /clear            → False  (清除历史+归档备份，继续)
        /history, /stats  → False  (显示消息数/文件大小/归档数)
        /archive          → False  (手动归档当前历史)
        /help             → False  (显示命令帮助)
        未知命令          → None   (交给 LLM 处理)

    chat.add_user_message(user_input)
    tool_runner.reset_count()
    round_usages = []  # 收集 token 用量

    # 内层循环: 处理 tool call 往返（上限 MAX_TOOL_ROUNDS=20）
    while tool_round < MAX_TOOL_ROUNDS:
        messages = chat.build_messages()         # system + history
        response = provider.chat(messages, tools)

        # 记录 token
        if provider.last_usage:
            round_usages.append(provider.last_usage)

        if tool_runner.has_tool_calls(response):
            chat.add_tool_call_message(...)       # SDK对象→dict
            results = tool_runner.execute(...)    # 权限→限流→执行→日志
            chat.add_tool_results(results)
            print(f"[工具调用: N个] [入 x (命中 n) | 出 x | 共 x]")
        else:
            print(f"助手: {response.content}")
            chat.add_assistant_message(...)
            break

    # 累计 token
    print(f"[Token: 输入 x (缓存命中 n) | 输出 x | 总计 x]")

    chat.save_history()   # 落盘历史（不含 system）
    chat.save_log(...)    # 落盘完整日志（含 system prompt + AGENT.md）
```

## 关键节点

| 节点 | 触发条件 | 行为 |
|------|----------|------|
| 历史裁剪 | `len(messages) >= history_max` | 保留后半，旧消息归档到 archive/ |
| 工具限流 | 超全局上限(100)或单工具上限(50) | 返回 `ERROR: ...上限已达` |
| 工具权限 | 命中 deny 或 enabled=false | 返回 `ERROR: ...已禁用` |
| API Key 校验 | 启动时 | 为空则抛出 ValueError 含设置指引 |
| 工具错误 | handler 返回 ERROR: | 直接交给 LLM 自行判断和调整 |
| 往返上限 | tool_round >= 20 | 跳出并告知用户重新描述需求 |
