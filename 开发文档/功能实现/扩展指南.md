# 扩展指南

最后更新：2026-05-04

## 添加 CLI 命令

位置: `main.py` → `_dispatch(cmd)`

返回值:
- `True`: 退出主循环
- `False`: 命令已处理
- `None`: 不是已知命令，交给 LLM

同步: README / AGENTS.md / 开发文档

## 添加 Web 命令

位置: `web/commands.py` → `_dispatch(cmd)`

返回 JSON dict。Web `/api/chat` 遇到 `/` 开头消息直接走命令分发。

## 添加 Web API

放入对应路由模块 (`web/routes/chat|files|conversations|system|config.py`)。

新建模块后在 `web/server.py` 末尾添加 import 触发路由注册:

```python
from web.routes import new_module  # noqa: E402,F401
```

同步: README / AGENTS.md / `开发文档/各模块进度/模块API.md`

## 修改对话循环

位置: `run/engine.py`

原则:
- CLI/Web 共用，不复制两套
- 保持事件类型兼容
- tool_calls 后必须写入对应 tool message
- DeepSeek thinking 模式下需从 response 提取 `reasoning_content` 传入 `chat.add_tool_call_message()`
- 保持 `MAX_TOOL_ROUNDS = 20`

## 修改 Provider

位置: `provider/openai_api.py`

注意:
- 不输出 API Key
- 保留 `last_usage`
- 保留流式/非流式兼容
- 保留 Windows `SSL_CERT_FILE` 防御

## 添加工具型 Skill

默认不要新增工具型 Skill。只有用户明确要求才加 `tool.py`。

```
skills/<name>/SKILL.md
skills/<name>/tool.py           # → register_tool(schema, handler)
```

## 添加指令型 Skill

推荐方式:

```
skills/<name>/SKILL.md          # name 与目录名一致，小写/数字/连字符
skills/<name>/scripts/          # 可选
skills/<name>/references/       # 可选
skills/<name>/assets/           # 可选
```

不要安装到 `users/<name>/skills/`。

## 安全代码位置

| 功能 | 文件 |
|---|---|
| 路径安全 | `skills/_common/__init__.py` |
| 文件操作 | `skills/file/tool.py` |
| 网络 SSRF | `skills/network/tool.py` |
| Shell 安全 | `skills/shell/tool.py` |
| 权限限流 | `run/tool.py` |
| Web 文件安全 | `web/routes/files.py` |

修改后验证: 路径越权、SSRF、shell 安全、deny 优先级、日志脱敏。
