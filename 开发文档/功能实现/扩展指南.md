# 扩展指南

## 添加新用户

```bash
cp -r users/kesepain users/<新用户名>
# 编辑 config.json → 修改 provider 配置
# 编辑 self_soul.md → 自定义角色人设
```

## 添加新 Skill（标准骨架）

遵循 agentskills.io 规范，创建目录结构：

```bash
skills/my-skill/
├── SKILL.md       # YAML frontmatter + Markdown 指引
└── tool.py        # register() 注册 tool_call
```

**SKILL.md 示例**:

```markdown
---
name: my-skill
description: 当用户需要 xxx 时使用此技能
---

# 我的技能

具体的使用指引...
```

**tool.py 示例**:

```python
# skills/my-skill/tool.py
from run.tool import register_tool
from skills._common import err, truncate

def my_func(param: str) -> str:
    try:
        return f"OK: {param}"
    except Exception as e:
        return err(f"失败: {e}")

SCHEMA = {
    "type": "function",
    "function": {
        "name": "my_func",
        "description": "功能说明",
        "parameters": {
            "type": "object",
            "properties": {
                "param": {"type": "string", "description": "参数说明"}
            },
            "required": ["param"]
        }
    }
}

def register():
    register_tool(SCHEMA, my_func)
```

放入 `skills/` 后，`register_all()` 自动发现 `SKILL.md` → 加载同目录 `tool.py` → 调用 `register()`。

**多文件 Skill**: 额外脚本放在 `scripts/` 或直接放在跟目录，只要不叫 `tool.py` 就不会被误扫。
参考文档放在 `references/`，模板放在 `assets/`。

## 更新 AGENT.md

添加新 Skill 后，同步更新 `AGENT.md` 中的 Skill 清单，确保 LLM 知道新能力。

## 禁用工具

```json
// 全局关闭: config/config_core.json
{"tool": {"enabled": {"delete_file": false}}}

// 用户黑名单: users/<name>/config.json
{"tool": {"deny": ["delete_file", "run_command"]}}
```

## 替换 Provider

`provider/` 下新建文件，实现：

```python
class YourProvider:
    def __init__(self, user_config: dict): ...
    def chat(self, messages, tools=None) -> Message: ...
```

`main.py` 替换 import 即可。返回的 Message 对象需有 `.content` 和 `.tool_calls` 属性。

## 预留目录

| 目录 | 规划用途 |
|------|----------|
| `agents/` | 不同 Agent 角色定义和配置 |
| `tools/` | 工具 JSON Schema 独立文件 |
| `prompt/` | 可复用 prompt 模板片段 |
