# 记忆与自改进 (Auto Improve)

管理用户的事实偏好、行为纠正规则和概念关系。数据存储在 `users/<name>/improve/`。

## 插件路径

`plugins/auto_improve/`

## 注册工具

| 工具 | 用途 | 典型触发 |
|------|------|----------|
| `auto_improve_save` | 保存永久记忆/规则/本体 | 用户说“记住”“保存” |
| `auto_improve_get` | 按键读取，永久层优先、临时层兜底 | 查询指定记忆 |
| `auto_improve_search` | 在全部类型和层级中搜索关键词 | 搜索历史偏好或规则 |
| `auto_improve_delete` | 删除指定永久记忆项 | 用户说“忘记”“删除记忆” |
| `auto_improve_list_pending` | 列出待审阅的临时文件 | 查看待审阅记忆 |
| `auto_improve_review` | 主动审阅临时内容并晋升有价值内容 | 用户说“审阅/整理记忆” |
| `auto_improve_review_log` | 查看近期审阅记录 | 用户问审阅了什么 |
| `auto_improve_cleanup` | 清理超过保留期的临时记忆 | 用户说清理过期记忆 |
| `auto_improve_cleanup_reviewed` | 删除已审阅吸收的临时文件 | 用户说清理已审阅内容 |

## 数据类型与参数

| `sub` 值 | 存储内容 | 示例 |
|----------|----------|------|
| `memory` | 用户事实、偏好、长期上下文 | 用户偏好简洁回复 |
| `self-improving` | 行为纠正、回复策略、改进规则 | 不要重复尝试同一失败命令 |
| `ontology` | 实体、关系、概念图谱 | 树莓派属于硬件设备 |

### 常用参数

| 工具 | 关键参数 |
|------|----------|
| `save` | `key` 文件键、`content` Markdown 内容、`sub` 数据类型 |
| `get` | `key`、`sub` |
| `search` | `query` 搜索词 |
| `delete` | `key`、`sub` |
| `cleanup` | `retention_days` 保留天数，默认 7 |
| `review_log` | `limit` 返回条数，默认 5 |

## 双阶段生命周期

### 被动模式

- 由消息阈值或后台流程触发。
- 可读取临时层已有内容用于去重。
- 只能写入临时层，禁止写入永久层。

### 主动审阅

- 由用户明确要求“审阅记忆”等触发。
- 读取临时层、永久层和当前对话。
- 将有长期价值的内容写入永久层，并记录 `review_log.jsonl`。
- 不直接修改临时文件；吸收后通过 `cleanup_reviewed` 清理。

## 结果说明

- 成功：返回保存路径、记忆内容、搜索结果、待审阅列表或审阅摘要。
- 未找到：返回明确的不存在提示。
- 失败：返回 `ERROR:` 前缀的错误信息。

## 常见规范

- 用户明确说“记住”时才直接写永久层。
- 不将 API Key、密码、令牌、身份证号等敏感信息写入长期记忆。
- `key` 使用稳定、可检索的英文或拼音名称，不含扩展名。
- 事实偏好放 `memory`，行为纠正放 `self-improving`，概念关系放 `ontology`。

## 常见处理办法

- **不确定记忆是否存在**：先 `search`，再按准确键 `get`。
- **用户要求整理记忆**：调用 `review`，不要逐条手工复制临时文件。
- **临时文件积压**：先审阅，再调用 `cleanup_reviewed`；未审阅旧内容用 `cleanup`。
- **用户要求忘记**：确认准确键和类型后调用 `delete`。

## 常见教训

- 被动模式不能碰永久层，主动审阅不能直接改临时层。
- `cleanup` 只清理临时层，不会自动删除永久记忆。
- 审阅不是无条件晋升：重复、短期、敏感或低价值内容应拒绝吸收。
- 删除永久记忆属于用户数据变更，必须由用户明确提出。
