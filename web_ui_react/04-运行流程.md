# 04 运行流程

最后更新：2026-05-06

本文档详细描述 CLI 和 Web 两种模式从启动到对话的完整链路。

---

## 一、CLI 运行流程

### 1.1 启动阶段

```
python start.py
  │
  ├─ 解析 sys.argv（无 --web 参数 → CLI 模式）
  │
  ├─ 扫描 users/ 目录，列出所有子目录
  │
  ├─ 用户选择（输入序号）
  │
  ├─ 设置环境变量 VOTX_USER_DIR = users/<name>/
  │
  └─ subprocess.run([python, main.py])  ← 子进程启动
       │
       └─ main.py 开始执行
```

### 1.2 初始化阶段

```
main.py
  │
  ├─ 读取 VOTX_USER_DIR 环境变量
  │
  ├─ 加载配置
  │   ├─ config/config_core.json → core_config
  │   └─ users/<name>/config.json → user_config
  │
  ├─ 创建 Provider
  │   └─ create_provider(user_config, core_config)
  │       └─ type="openai" → ResponsesProvider
  │       └─ type="anthropic" → AnthropicProvider
  │
  ├─ 构建 system prompt
  │   └─ build_cached_system_prompt(root, user_dir)
  │       └─ 内部调用 build_system_prompt(root, user_dir)
  │           └─ 8 层拼接（见下方 system prompt 构建）
  │           └─ 内部调用 register_all() 注册所有 Skill
  │
  ├─ 加载工具 schema
  │   └─ load_tool_schemas()  ← 必须在 build_system_prompt 之后
  │
  ├─ 初始化 ChatManager
  │   ├─ 设置 system_prompt
  │   └─ load_history()  ← 从 JSON 恢复历史 + tool_calls 链修复
  │
  ├─ 初始化 ToolRunner（权限 + 限流参数）
  │
  ├─ 注册 atexit 退出钩子（保存历史 + 生成摘要）
  │
  └─ 打印就绪信息 + 命令列表
```

### 1.3 对话循环

```
while True:
  │
  ├─ input("您: ")  ← 等待用户输入
  │
  ├─ 空输入 → continue
  │
  ├─ "/" 开头 → _dispatch(cmd)
  │   ├─ /exit /quit /q  → 生成摘要 + 保存 + break
  │   ├─ /clear           → 归档 + 清空历史 + 清空工具日志
  │   ├─ /retry           → 移除最后一条 AI 回复 + 重新发送
  │   ├─ /history /stats  → 打印历史状态
  │   ├─ /archive         → 手动归档 + 同步摘要索引
  │   ├─ /summarize       → 生成对话摘要
  │   ├─ /help            → 打印帮助
  │   └─ None             → 交给 LLM 处理
  │
  └─ _run_turn(user_text)
      │
      ├─ chat.add_user_message(text)
      ├─ tool_runner.reset_count()
      │
      ├─ for event in run_chat_turn(chat, tool_runner, provider, tools):
      │   ├─ tool_call   → 打印工具调用行 (📖 read_file /path 0.9s)
      │   ├─ text_chunk  → 实时打印正文
      │   ├─ text_done   → 换行
      │   ├─ text        → 打印完整回复（非流式）
      │   ├─ usage       → 收集 token 统计
      │   ├─ error       → 打印错误
      │   ├─ deadlock_warning → 打印警告
      │   └─ max_rounds  → 打印上限提示
      │
      ├─ 打印 token 统计摘要
      │
      └─ chat.save_history() + chat.save_log()
```

## 二、Web 运行流程

### 2.1 启动阶段

```
python start_web.py（或 python start.py --web）
  │
  ├─ 解析 --port 参数（默认 1478 / 13579）
  │
  ├─ 端口冲突轮询（最多 10 次，逐个尝试）
  │
  ├─ _check_users()
  │   ├─ 有用户 → 继续
  │   └─ 无用户 → 引导 set_user.py add_user() 创建
  │
  └─ run_server(port)
      │
      ├─ Flask app 创建（secret_key 从 .session_secret 读取或自动生成）
      │
      ├─ 路由注册（导入 web/routes/*.py 触发 @app.route 装饰器）
      │   ├─ chat.py         → 7 个端点
      │   ├─ config.py       → 2 个端点
      │   ├─ conversations.py → 8 个端点
      │   ├─ files.py        → 6 个端点
      │   └─ system.py       → 6 个端点
      │
      └─ app.run(host="0.0.0.0", port=port, threaded=True)
```

### 2.2 用户选择阶段

```
浏览器访问 http://localhost:1478
  │
  ├─ GET / → 渲染 index.html (Vue 3 单页应用)
  │
  ├─ 前端调用 GET /api/users
  │   └─ 扫描 users/ 目录，返回用户列表
  │
  ├─ 用户点击选择
  │   └─ POST /api/select-user {user: "name"}
  │       ├─ 读取 config_core.json + users/<name>/config.json
  │       ├─ init_user_session()
  │       │   ├─ create_provider(user_config, core_config)
  │       │   ├─ ChatManager + load_history()
  │       │   ├─ build_cached_system_prompt() + register_all()
  │       │   ├─ load_tool_schemas()
  │       │   ├─ ToolRunner
  │       │   └─ 存入 _sessions[user_name]
  │       └─ flask_session["user_name"] = user_name
  │
  └─ 前端调用 GET /api/session → 确认会话已激活
```

### 2.3 对话流程

```
用户在输入框输入消息，点击发送
  │
  ├─ POST /api/chat {message: "用户消息"}
  │   │
  │   ├─ 检查 session（未选择用户 → 400）
  │   │
  │   ├─ "/" 开头 → _dispatch(cmd)
  │   │   └─ 返回 JSON（非流式命令结果）
  │   │
  │   ├─ chat.add_user_message(message)
  │   ├─ tool_runner.reset_count()
  │   │
  │   └─ SSE 流式响应（text/event-stream）
  │       │
  │       ├─ for event in run_chat_turn(chat, tool_runner, provider, tools):
  │       │   └─ yield f"data: {json.dumps(event)}\n\n"
  │       │
  │       ├─ finally:
  │       │   ├─ chat.save_history()
  │       │   └─ chat.save_log(chat.build_messages())
  │       │
  │       └─ yield "data: {type: done}\n\n"
  │
  └─ 前端逐条接收 SSE 事件并渲染
      ├─ thinking_chunk → 思考区域逐字显示
      ├─ thinking_done  → 思考区域折叠
      ├─ text_chunk     → 正文区域逐字显示
      ├─ text_done      → 正文完成
      ├─ tool_call      → 工具调用卡片（图标 + 名称 + 参数 + 耗时）
      ├─ usage          → token 统计显示
      ├─ error          → 错误提示
      └─ done           → 完成，刷新右侧面板
```

### 2.4 其他 Web 操作

```
断开连接:
  POST /api/disconnect
    → _web_summarize() 生成摘要
    → chat.save_history() + save_log()
    → clear_session()
    → 清除 flask_session

配置修改:
  POST /api/config {model, api_key, base_url, ...}
    → 保存到 config.json
    → 关键字段变更 → 重建 Provider
    → 非关键字段 → setattr 更新内存属性

动态重载:
  POST /api/reload
    → 清空 TOOL_REGISTRY → register_all() 重载 Skill
    → invalidate_prompt_cache → 重建 system prompt
    → 重建 ToolRunner

归档继续:
  POST /api/conversations/continue {id: "history_xxx.json.gz"}
    → _validate_conv_id() 安全校验
    → 自动归档当前对话
    → 读取目标归档消息
    → chat.load_messages() + save_history()
```

## 三、system prompt 构建流程

```
build_system_prompt(root, user_dir)
  │
  ├─ 1. users/<name>/self_soul.md           用户人设（最外层，优先级最高）
  │
  ├─ 2. config/soul.md                      全局人格基座（非空非注释时注入）
  │
  ├─ 3. AGENTS.md                           自我操作手册（Agent 执行规则）
  │
  ├─ 4. skills.register_all()               Skill 摘要
  │   ├─ 工具型 Skill（可 function call）
  │   └─ 指令型 Skill（正文引导）
  │
  ├─ 5. users/<name>/self-improving/memory.md    HOT Tier 记忆
  │
  ├─ 6. users/<name>/self-improving/corrections.md  纠正记录
  │
  ├─ 7. users/<name>/memory/*.md            长期记忆（单文件限 2000 字符）
  │
  ├─ 8. memory/ontology/graph.jsonl         知识图谱摘要（仅实体数）
  │
  └─ 9. SESSION-STATE.md                    会话状态（Hot RAM，可选）
```

缓存策略：`prompt_cache.py` 按上述所有源文件的 mtime 计算 hash，文件不变时直接返回缓存。

## 四、tool_calls 执行循环

```
run_chat_turn(chat, tool_runner, provider, tools)
  │
  ├─ tool_round = 0（最大 MAX_TOOL_ROUNDS，默认 80）
  │
  ├─ while tool_round < MAX_TOOL_ROUNDS:
  │   │
  │   ├─ messages = chat.build_messages()
  │   │
  │   ├─ 流式路径:
  │   │   ├─ for item in provider.respond_stream(messages, tools):
  │   │   │   ├─ thinking_chunk → yield
  │   │   │   └─ text_chunk → yield
  │   │   └─ response = provider.last_response
  │   │
  │   ├─ 非流式路径:
  │   │   └─ response = provider.respond(messages, tools)
  │   │
  │   ├─ 有 tool_calls:
  │   │   ├─ chat.add_tool_call_message(tool_calls, reasoning)
  │   │   ├─ results, details = tool_runner.execute(response)
  │   │   │   └─ 对每个 tool_call:
  │   │   │       ├─ _check_limit(name) — deny > enabled > 上限
  │   │   │       ├─ ThreadPoolExecutor + timeout 执行 handler
  │   │   │       ├─ log_tool_call() — 写入 tool_log.jsonl
  │   │   │       └─ 返回 tool message
  │   │   ├─ chat.add_tool_results(results)
  │   │   ├─ tool_round += 1
  │   │   ├─ yield tool_call 事件
  │   │   └─ 死循环检测（同命令连败 3 次 → yield deadlock_warning）
  │   │
  │   └─ 无 tool_calls:
  │       ├─ chat.add_assistant_message(reply, reasoning)
  │       ├─ yield text / text_done
  │       └─ return（对话结束）
  │
  └─ 达到 MAX_TOOL_ROUNDS:
      ├─ chat.add_assistant_message("已达到最大工具调用轮数")
      └─ yield max_rounds
```

## 五、历史管理流程

### 5.1 保存

```
chat.save_history()
  │
  ├─ _close_tool_chain()
  │   └─ 末尾如果是未闭合的 assistant(tool_calls)
  │       → 补上 tool 错误消息 "ERROR: 程序中断，工具调用未完成"
  │
  └─ atomic_write_json(data_path, messages, indent=2)
      └─ 临时文件 → flush → fsync → os.replace（原子写）
```

### 5.2 加载

```
chat.load_history()
  │
  ├─ read_json_safe(data_path)
  │   ├─ 正常 → messages
  │   └─ 损坏 → 备份为 .corrupt，messages = []
  │
  └─ _repair_tool_chain()
      ├─ 第一遍：收集所有 assistant(tool_calls) 声明的 ID
      └─ 第二遍：顺序扫描
          ├─ tool 消息 ID 不在 valid_ids 或未见过 → 切除
          └─ assistant(tool_calls) 后缺少对应 tool → 切除
```

### 5.3 自动裁剪

```
_trim_if_needed()
  │
  ├─ len(messages) < history_max → 不裁剪
  │
  ├─ 找安全切点（不在 tool_calls/tool 配对中间）
  │
  ├─ 提取旧消息摘要（首尾各 3 条）
  │
  ├─ 插入摘要消息到历史开头
  │
  └─ 归档旧消息
      ├─ zip_history=True → gzip 压缩 (.json.gz)
      └─ zip_history=False → 纯 JSON
```

## 六、SSE 事件类型汇总

| 事件类型 | 触发时机 | 数据格式 |
|---|---|---|
| `thinking_chunk` | 流式思考片段 | `{type, content}` |
| `thinking_done` | 流式思考结束 | `{type}` |
| `thinking` | 非流式思考块 | `{type, content}` |
| `text_chunk` | 流式正文片段 | `{type, content}` |
| `text_done` | 流式正文结束 | `{type}` |
| `text` | 非流式正文 | `{type, content}` |
| `tool_call` | 单次工具调用完成 | `{type, name, icon, args, elapsed, success, line}` |
| `usage` | Token 用量 | `{type, data: {prompt_tokens, completion_tokens, cached_tokens, total_tokens, elapsed}}` |
| `error` | Provider 错误 | `{type, content}` |
| `deadlock_warning` | 同命令连败 3 次 | `{type}` |
| `max_rounds` | 达到最大工具轮数 | `{type}` |
| `done` | 对话完成 (Web) | `{type}` |
