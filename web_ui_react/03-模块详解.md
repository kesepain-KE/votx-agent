# 03 模块详解

最后更新：2026-05-06

本文档覆盖所有模块的概述和具体函数功能。

## 一、入口层

### 1.1 `votx.py` — 入口命令

统一入口命令，分发到 CLI 或 Web。

| 函数 | 功能 |
|---|---|
| `show_help()` | 打印帮助信息（子命令列表） |
| `main()` | 解析 `sys.argv`，分发到 `start.py`（CLI）或 `start_web.py`（Web） |

子命令：`votx` / `votx web` → Web UI，`votx cli` → CLI，`votx help` → 帮助。

### 1.2 `start.py` — CLI / Web 双模式入口

| 函数 | 功能 |
|---|---|
| `main_cli()` | 扫描 `users/` → 列出用户 → 选择 → `subprocess` 启动 `main.py` |
| `main_web()` | 解析 `--port` 参数 → 导入 `web.server.run_server()` 启动 Flask |

CLI 模式通过 `subprocess.run` 启动 `main.py`，将 `VOTX_USER_DIR` 写入环境变量。

### 1.3 `start_web.py` — Web 专用入口

| 函数 | 功能 |
|---|---|
| `_check_users()` | 检查是否存在至少一个用户，无则引导 `set_user.py add_user()` 创建 |
| 顶层逻辑 | 解析 `--port` → 端口冲突自动轮询（最多 10 次）→ `_check_users()` → `run_server()` |

### 1.4 `main.py` — CLI 主循环

| 函数 | 功能 |
|---|---|
| `main()` | CLI 主入口。初始化 Provider / ToolRunner / ChatManager → 对话循环 |
| `_run_turn(user_text)` | 执行一轮对话：`add_user_message` → `reset_count` → 消费 `run_chat_turn` 事件流 → 打印输出 → 保存历史 |
| `_dispatch(cmd)` | CLI 命令分发。返回 `True`=退出、`False`=已处理、`None`=交给 LLM |

CLI 支持的命令：`/exit` `/quit` `/q` `/clear` `/retry` `/history` `/stats` `/archive` `/summarize` `/summary` `/总结` `/help`

退出时通过 `atexit.register(_on_exit)` 自动保存历史和生成摘要。

### 1.5 `set_user.py` — 用户管理

| 函数 | 功能 |
|---|---|
| `list_users()` | 返回已配置用户列表（有 `config.json` 的目录） |
| `add_user(name)` | 交互式创建用户：目录结构 + config.json + self_soul.md + self-improving 模板 |
| `edit_user(name)` | 编辑已有用户：模型 / API Key / Base URL / 思考模式 / 流式输出 |
| `menu()` | 交互式菜单循环 |
| `_ensure_dirs(user_dir)` | 创建完整用户目录结构（history/chat/log/archive/file、download、memory、self-improving） |
| `_pick_model(default)` | 交互式模型选择（预设 + 自定义） |

### 1.6 `paths.py` — 路径解析

| 函数 | 功能 |
|---|---|
| `get_project_root()` | 返回项目根目录。三种模式通用：`sys.frozen` → PyInstaller --onedir/--onefile，否则 `__file__` 目录 |

## 二、Provider 层 (`provider/`)

### 2.1 `schema.py` — 统一数据结构

| 类型 | 字段 | 说明 |
|---|---|---|
| `ToolCall` | `id`, `name`, `input` | 统一工具调用，`input` 是已解析的 dict |
| `ProviderResponse` | `text`, `reasoning`, `tool_calls`, `usage`, `finish_reason` | 统一 LLM 响应 |
| `ProviderResponse.has_tool_calls` | property | 是否包含工具调用 |

### 2.2 `base.py` — 抽象接口

| 方法 | 说明 |
|---|---|
| `respond(messages, tools)` | 非流式调用，返回 `ProviderResponse` |
| `respond_stream(messages, tools)` | 流式调用，yield 事件 dict。生成器耗尽后 `self.last_response` 可用 |

属性：`last_response`、`last_usage`、`stream`

### 2.3 `factory.py` — Provider 工厂

| 函数 | 功能 |
|---|---|
| `create_provider(user_config, core_config)` | 根据 `provider.type` 创建对应实例。`openai` → `ResponsesProvider`，`anthropic` → `AnthropicProvider` |

支持 `VOTX_PROVIDER` 环境变量覆盖 type。兼容旧值：`deepseek`、`openai_compatible`、`azure_openai`、`google_gemini` 均走 `ResponsesProvider`。

### 2.4 `openai_api.py` — OpenAI Chat Completions Provider

类 `DeepSeekProvider(BaseProvider)`：

| 方法 | 功能 |
|---|---|
| `__init__(user_config, core_config)` | 初始化 OpenAI 客户端。API Key 来源：config > `DEEPSEEK_API_KEY` > `OPENAI_API_KEY`。base_url 来源：config > `DEEPSEEK_BASE_URL` > 默认 |
| `respond(messages, tools)` | 非流式调用，支持超时重试（APITimeoutError / APIConnectionError 最多重试 2 次） |
| `respond_stream(messages, tools)` | 流式调用，yield `thinking_chunk` / `text_chunk` 事件。内部拼接 tool_calls delta |
| `_collect_stream(response)` | 收集流式 chunk 拼接为完整 `ChatCompletionMessage` |

模块级函数：

| 函数 | 功能 |
|---|---|
| `_load_dotenv()` | 手动解析 `.env` 文件写入 `os.environ`（不依赖 python-dotenv） |
| `_to_provider_response(msg)` | `ChatCompletionMessage` → `ProviderResponse` |
| `_extract_usage(response)` | 提取 token 用量（含 DeepSeek 缓存命中 `cached_tokens`） |

### 2.5 `responses_api.py` — OpenAI Responses API Provider

类 `ResponsesProvider(BaseProvider)`：

| 方法 | 功能 |
|---|---|
| `__init__(user_config, core_config)` | 初始化。`api_style="responses"` 强制 Responses API，否则走 Chat Completions |
| `respond(messages, tools)` | 优先 Responses API，不支持时自动回退 Chat Completions |
| `respond_stream(messages, tools)` | 同上，流式版本 |
| `_respond_via_responses(messages, tools)` | Responses API 非流式路径 |
| `_respond_stream_via_responses(messages, tools)` | Responses API 流式路径，yield `text_chunk` / `thinking_chunk` |
| `_respond_via_chat(messages, tools)` | Chat Completions 回退路径（创建临时 `DeepSeekProvider`） |
| `_respond_stream_via_chat(messages, tools)` | Chat Completions 流式回退路径 |

模块级函数：

| 函数 | 功能 |
|---|---|
| `_to_responses_input(messages)` | 内部 dict 格式 → Responses API `input` + `instructions` |
| `_to_responses_tools(tools)` | Chat Completions tool schema → Responses API 扁平格式 |
| `_from_responses_output(response)` | Responses API 响应 → `ProviderResponse` |
| `_extract_responses_usage(response)` | 提取 token 用量（`input_tokens` / `output_tokens`） |
| `_is_not_supported(error)` | 判断是否 404/405 不支持错误 |

### 2.6 `anthropic_adapter.py` — Anthropic Messages API

类 `AnthropicProvider(BaseProvider)`：

| 方法 | 功能 |
|---|---|
| `__init__(user_config, core_config)` | 初始化 Anthropic 客户端。支持 `extended thinking` 配置 |
| `respond(messages, tools)` | 非流式调用 Anthropic Messages API |
| `respond_stream(messages, tools)` | 流式调用，处理 `content_block_start` / `content_block_delta` / `message_delta` 事件 |

模块级函数：

| 函数 | 功能 |
|---|---|
| `_to_anthropic_messages(messages)` | 内部 dict → Anthropic 格式（system prompt 提取为顶层参数） |
| `_to_anthropic_tools(tools)` | OpenAI tool schema → Anthropic 格式（`input_schema`） |
| `_from_anthropic_response(response)` | Anthropic 响应 → `ProviderResponse`（解析 text / thinking / tool_use block） |
| `_extract_anthropic_usage(response)` | 提取 token 用量（含 `cache_read_input_tokens`） |

## 三、引擎层 (`run/`)

### 3.1 `engine.py` — 对话引擎

| 函数 | 功能 |
|---|---|
| `build_system_prompt(root, user_dir)` | 组装完整 system prompt（8 层拼接，详见运行流程文档） |
| `run_chat_turn(chat, tool_runner, provider, tools)` | 对话循环生成器。流式/非流式统一处理，tool_calls 多轮循环，yield 事件 dict |
| `_fmt_tool_line(name, args, elapsed, success)` | 格式化工具调用显示行（图标 + 名称 + 参数 + 耗时） |
| `_pick_arg(name, args)` | 提取展示用的关键参数（截断长字符串） |
| `_load_max_tool_rounds()` | 从 `config_core.json` 读取工具轮数上限 |

事件类型：
```
thinking_chunk / thinking_done / thinking    思考内容
text_chunk / text_done / text                正文内容
tool_call                                    工具调用详情
usage                                        Token 用量
error                                        错误信息
deadlock_warning                             同命令连败 3 次警告
max_rounds                                   达到最大工具轮数
done                                         对话完成 (Web)
```

死循环检测：同一工具+参数连续失败 3 次，注入提示让 LLM 换思路。

### 3.2 `chat.py` — 对话管理

类 `ChatManager`：

| 方法 | 功能 |
|---|---|
| `__init__(user_dir, core_config, user_config)` | 初始化路径（data_path / log_path / tool_log_path / archive_dir）和 token 统计 |
| `set_system_prompt(prompt)` | 设置 system prompt |
| `build_messages()` | 组装完整消息列表：system + history |
| `add_user_message(content)` | 追加用户消息（自动裁剪） |
| `add_assistant_message(content, reasoning)` | 追加助手消息 |
| `add_tool_call_message(tool_calls, reasoning)` | 将 tool_calls 转为可序列化 dict 后追加 |
| `add_tool_results(tool_outputs)` | 追加工具结果消息 |
| `accumulate_usage(usage, is_tool_round, elapsed_ms)` | 累加 token 统计（跨轮会话累加） |
| `load_history()` | 从 JSON 文件恢复历史（损坏时自动修复 + tool_calls 链检查） |
| `load_messages(messages)` | 替换当前消息列表并修复 tool_calls 链 |
| `save_history()` | 落盘历史（不含 system，保存前自动闭合未完成 tool_calls 链） |
| `save_log(full_messages)` | 落盘完整日志（含 system prompt） |
| `clear_history()` | 清除当前对话（先归档再清空，同时清空工具日志） |
| `archive_now()` | 手动归档全部历史（不清空） |
| `history_stats()` | 返回历史状态摘要（消息数 / 文件大小 / 工具日志数 / 归档数） |
| `_repair_tool_chain()` | 修复历史中 tool_calls 链断裂（前向+后向+乱序检查） |
| `_close_tool_chain()` | 末尾未完成 tool_calls 补充错误 tool message |
| `_trim_if_needed()` | 超过上限时裁剪，安全切点（不在 tool_calls/tool 配对中间截断），插入摘要消息 |
| `_archive(old_messages)` | 归档旧消息到独立文件（支持 gzip 压缩） |
| `_extract_digest(messages, max_items)` | 从消息列表提取关键对话摘要 |

### 3.3 `tool.py` — 工具执行

| 函数/方法 | 功能 |
|---|---|
| `register_tool(schema, handler, meta)` | 注册工具到全局 `TOOL_REGISTRY` |
| `load_tool_schemas()` | 返回排序后的 schema 列表（OpenAI tools 格式） |
| `clear_tool_registry()` | 清空全局注册表（reload 前调用） |
| `get_tool_registry_snapshot()` | 返回注册表浅拷贝 |

类 `ToolRunner`：

| 方法 | 功能 |
|---|---|
| `__init__(core_config, user_config)` | 初始化限流参数（全局/单工具上限）和权限（deny/enabled） |
| `has_tool_calls(response)` | 检查响应是否含工具调用 |
| `execute(response)` | 执行所有 tool_calls。流程：权限检查 → 限流检查 → 超时执行（ThreadPoolExecutor） → 日志 → 返回结果 |
| `reset_count()` | 重置调用计数器 |
| `_check_limit(name)` | 返回错误字符串或 None（deny > enabled > 全局上限 > 单工具上限） |
| `_count(name)` | 递增调用计数 |

### 3.4 `summarize.py` — 摘要生成

| 函数 | 功能 |
|---|---|
| `generate_summary(provider, messages)` | 子代理：用独立 LLM 调用生成一句话摘要（不超过 30 字） |
| `summarize_and_store(provider, messages, user_dir)` | 生成摘要并存入索引文件 |
| `load_index(user_dir)` | 加载摘要索引 (`history_save_data.json`) |
| `save_index(user_dir, index)` | 保存摘要索引 |
| `sync_to_new_archives(user_dir, before)` | 将当前摘要同步到新生成的归档文件 |
| `index_path(user_dir)` | 返回索引文件路径 |

### 3.5 `io_utils.py` — IO 工具

| 函数 | 功能 |
|---|---|
| `atomic_write_text(path, text)` | 原子写文本：临时文件 → flush → fsync → `os.replace` |
| `atomic_write_json(path, data, indent)` | 原子写 JSON |
| `atomic_write_gzip(path, text)` | 原子写 gzip 文件 |
| `append_jsonl(path, entry)` | 追加一行 JSONL |
| `read_json_safe(path, default)` | 安全读 JSON（损坏返回 default） |

### 3.6 `prompt_cache.py` — 缓存管理

| 函数 | 功能 |
|---|---|
| `build_cached_system_prompt(root, user_dir, force)` | 缓存版 system prompt 构建。按 mtime 计算缓存 key，命中则直接返回 |
| `invalidate_prompt_cache(user_dir)` | 使缓存失效（指定用户或全部） |
| `get_prompt_cache_key(root, user_dir)` | 获取当前缓存 key |
| `_compute_cache_key(root, user_dir)` | 根据所有源文件 mtime 计算缓存 key |
| `_mtime(path)` | 文件 mtime 纳秒，不存在返回 -1 |
| `_dir_max_mtime(directory, pattern)` | 目录下匹配文件的最大 mtime |

缓存源文件包括：`self_soul.md`、`soul.md`、`AGENTS.md`、所有 `SKILL.md`/`tool.py`/`_meta.json`、`self-improving/`、`memory/*.md`、`SESSION-STATE.md`、`graph.jsonl`

| 函数/逻辑 | 功能 |

## 五、Skill 层 (`skills/`)

### 5.1 `skills/__init__.py` — Skill 自动发现

| 函数 | 功能 |
|---|---|
| `register_all(force_reload, clear_registry)` | 扫描所有 `SKILL.md` → 有 `tool.py` 则注册工具 → 收集指令 Skill 摘要 → 返回指令列表 |
| `_parse_skill_md(path)` | 解析 SKILL.md 的 YAML frontmatter，返回 `(name, body)` |

加载流程：
1. 清空 `TOOL_REGISTRY`
2. 将每个 skill 目录加入 `sys.path`
3. 遍历所有 `SKILL.md`：有 `tool.py` → `importlib` 动态加载 → 调用 `register()`
4. 收集摘要行（name + type + description）

### 5.2 工具型 Skill 清单（12 个）

| Skill | 注册工具 | 功能 |
|---|---|---|
| `file` | `read_file` `write_file` `list_dir` `delete_file` | 文件读写/列出/删除 |
| `network` | `http_get` `http_post` | HTTP 请求（含 SSRF 防护） |
| `shell` | `run_command` | Shell 命令执行（含危险命令黑名单） |
| `time` | `get_time` `sleep` | 时间查询 / 延时 |
| `tavily-search` | `tavily_search` | Tavily 网络搜索 |

```json
    "history": {
        "history_max": 100,        // 会话消息上限

        "tool_max_per_type": 80,    // 单轮工具调用总上限

        "tool_log_max": 1000,       // 工具日志上限（预留）
        "enabled": {}               // 全局工具白名单
    },
    "output": {
        "stream": true              // 默认流式输出
    }
}
```
