## When to Use

用户纠正或指出错误时。完成重要工作后评估结果。发现输出可以更好时。知识应随时间累积而无需手动维护。

## 架构

记忆存储于 `users/<name>/self-improving/`，分层结构。

安装后首次激活时自动创建目录结构：

```
users/<name>/self-improving/
├── memory.md          # HOT: ≤100 行，始终加载
├── corrections.md     # 最近 50 条纠正记录
├── projects/          # 项目级学习
├── domains/           # 领域特定（代码、写作、沟通）
└── archive/           # COLD：废弃的模式
```

## 学习信号

发现以下模式时自动记录：

**纠正** → 追加到 `corrections.md`，评估是否提升到 `memory.md`：
- "不对，不是这样..."
- "实际上是..."
- "你搞错了..."
- "我喜欢 X，不是 Y"
- "记住我历来..."
- "我之前告诉过你..."
- "别再做 X"
- "你怎么老是..."

**偏好信号** → 明确时直接写入 `memory.md`：
- "我喜欢你这样做..."
- "每次都为我做 X"
- "永远不要做 Y"
- "我的风格是..."
- "对于[项目]，用..."

**模式候选** → 跟踪，3 次后提升：
- 同一指令重复 3 次以上
- 反复有效的工作流程
- 用户表扬某个具体做法

**忽略**（不记录）：
- 一次性指令（"现在做 X"）
- 上下文特定的（"在这个文件里..."）
- 假设性的（"如果..."）

## 自我反思

完成重要工作后，暂停评估：

1. **达到预期了吗？** — 比较结果 vs 意图
2. **什么是可以改进的？** — 为下次找改进点
3. **这是模式吗？** — 如果是，记录到 `corrections.md`

**何时反思：**
- 完成多步骤任务后
- 收到反馈后（正面或负面）
- 修复 bug 或错误后
- 发现输出可以更好时

**记录格式：**
```
CONTEXT: [任务类型]
REFLECTION: [注意到什么]
LESSON: [下次怎么做]
```

自我反思条目同样遵循提升规则：成功应用 3 次 → 提升到 HOT。

## 核心规则

### 1. 从纠正和自我反思中学习
- 用户明确纠正时记录
- 发现自己可改进时记录
- 从不单凭沉默推断
- 相同教训 3 次后 → 问用户是否确认为规则

### 2. 分层存储
| 层级 | 位置 | 大小限制 | 行为 |
|------|------|----------|------|
| HOT | `memory.md` | ≤100 行 | 始终加载 |
| WARM | `projects/` `domains/` | 每文件 ≤200 行 | 上下文匹配时加载 |
| COLD | `archive/` | 无限 | 明确查询时加载 |

### 3. 自动提升/降级
- 7 天内使用 3 次的模式 → 提升到 HOT
- 30 天未使用 → 降级到 WARM
- 90 天未使用 → 归档到 COLD
- 未经许可绝不删除

### 4. 命名空间隔离
- 项目模式 → `projects/{name}.md`
- 全局偏好 → HOT 层（`memory.md`）
- 领域模式（代码、写作）→ `domains/`
- 交叉命名空间继承：全局 → 领域 → 项目

### 5. 冲突解决
模式冲突时：
1. 最具体的优先（项目 > 领域 > 全局）
2. 最新的优先（同级）
3. 模棱两可 → 问用户

### 6. 压缩
文件超限时：
1. 合并相似纠正为单条规则
2. 归档未使用的模式
3. 总结冗长的条目
4. 从不丢失已确认的偏好

### 7. 透明性
- 每次使用记忆 → 注明来源："使用 X（来自 projects/foo.md:12）"
- 定期摘要可选
- 可随时完全导出

### 8. 安全边界
绝不存储：凭据、财务数据、医疗信息、生物特征、第三方信息、位置模式、访问模式。

### 9. 优雅降级
上下文限制达到时：
1. 只加载 `memory.md`（HOT）
2. 按需加载相关命名空间
3. 从不静默失败 — 告知用户什么没加载

## 数据存储

本地状态位于 `users/<name>/self-improving/`：

- `memory.md` — HOT 规则和已确认偏好
- `corrections.md` — 明确纠正和可复用的教训
- `projects/` 和 `domains/` — 限定范围的模式
- `archive/` — 废弃或不活动的模式

## 首次启动

自动创建目录结构和初始文件。无需手动设置。
