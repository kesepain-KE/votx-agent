# Memory Operations (votx-agent 适配版)

存储路径：`users/<name>/self-improving/`

## 用户命令

| 命令 | 操作 |
|------|------|
| "关于 X 你知道什么？" | 搜索所有层级，返回匹配结果与来源 |
| "显示我的记忆" | 显示 memory.md 内容 |
| "显示[项目]的模式" | 加载并显示特定命名空间 |
| "忘记 X" | 从所有层级移除，确认删除 |
| "忘记一切" | 完全清空（先导出选项） |
| "最近有什么变化？" | 显示最近 20 条纠正 |
| "导出记忆" | 打包所有记忆文件 |
| "记忆状态" | 显示各层级大小和健康状态 |

## 自动操作

### 会话开始时
1. 加载 `memory.md`（HOT 层）
2. 如有上下文提示，预加载相关命名空间

### 收到纠正时
1. 解析纠正类型（偏好、模式、覆盖）
2. 检查是否重复
3. 如新：追加到 `corrections.md`，递增计数
4. 如重复：更新计数，3 次后确认提升为规则
5. 确定命名空间（全局、领域、项目）
6. 写入对应文件

### 模式匹配时
1. 查找模式来源（file:line）
2. 应用模式
3. 注明来源

### 定期维护
1. 扫描所有文件查找降级候选
2. >30 天未用 → 降级到 WARM
3. >90 天未用 → 归档到 COLD
4. 文件超限时压缩
5. 生成摘要（可选）

## 边界情况处理

### 检测到冲突
用户说 "用 tab"，又说 "用空格" → 项目级覆盖全局级，记录冲突。

### 用户改变主意
旧偏好归档，新偏好设为待定。

### 上下文不明确
问用户："这是全局适用还是仅当前项目？"
