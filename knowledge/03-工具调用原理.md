# 工具调用原理

更新时间：2026-07-12

## Skill 类型与注册

| 类型 | 文件结构 | 进入模型的方式 |
|---|---|---|
| 工具型 | `SKILL.md` + `tool.py` | schema 注册到 `TOOL_REGISTRY`，模型通过 function call 调用 |
| 指令型 | `SKILL.md` | 摘要注入 System Prompt，模型按流程调用已有工具 |

启动和刷新 Prompt 时，`skills.register_all()` 扫描内置 `plugins/` 与用户 `skills/`，调用每个工具型 Skill 的 `register()`。`run.tool.register_tool(schema, handler, meta)` 将 schema、handler 和可选 meta 写入全局注册表。

用户 Skill 可通过元数据覆盖同名内置 Skill。`skills.disabled_builtin` 会同时影响 schema 列表和 ToolRunner 执行检查。

## 调用循环

`run.engine.run_chat_turn()` 的主要流程：

1. `chat.build_messages()` 构建 System Prompt 与历史。
2. Provider 返回文本或 `tool_calls`。
3. `ToolRunner.execute()` 解析工具名和参数。
4. 检查用户/全局 JSON 配置中的工具开关、deny 和禁用 Skill。
5. 在线程池中执行 handler，并按 `tool_timeout` 等待。
6. 写入工具日志，生成 tool role 消息和前端详情。
7. 工具结果回填历史，继续下一轮 Provider 请求。
8. 无工具调用时结束并保存最终回复。

工具可通过事件队列推送中间进度。`time.sleep` 使用 `skip_tool_timeout` meta，由工具自身的上限控制。

## ToolRunner 配置

配置来源：

```text
users/<name>/config.json > config/config_core.json > 代码默认值
```

当前有效控制：

- `tool.enabled`
- 用户 `tool.deny`（优先）
- `skills.disabled_builtin`
- `tool.tool_timeout`
- 用户取消执行

`tool.tool_max_per_type` 与 `tool.tool_max_per_tool` 字段仍存在于全局配置，但当前 `_check_limit()` 已移除硬编码计数拦截，只保留配置级权限检查。不要把配置文件中的数字误写成当前正在生效的调用次数限制。

## 当前内置插件边界

现有 20 个内置插件目录：

```text
audio_universal  auto_improve       download_anything
file             image_edit         image_generation
kb_retriever     network            qq_file
qq_send          shell              skill_creator
speech_generation speech_to_speech  task_plan
task_time        tavily_search      time
video_generation vision_universal
```

其中 `kb_retriever`、`skill_creator` 为指令型，其余为工具型。

当前源码不包含：

```text
旧版文档转换、PDF 与 DOCX 插件
```

旧收口关系：

| 旧插件 | 当前归属 |
|---|---|
| `file_search` | `file.search_files` |
| `video_download` | `download_anything.download_video` |
| `web_content_fetcher` | `network.web_read` |

## 路径、命令和网络现状

- `safe_path()` 只负责把相对路径按项目根解析为绝对路径，不做目录白名单。
- file 工具保留存在/类型检查、编码回退和文件操作错误提示。
- shell 支持 cwd、env、stdin、命令链和可选 session，不含旧危险命令黑名单。
- network 提供 HTTP GET/POST 和网页正文读取，不含旧 `network_scope`/SSRF 地址过滤。
- download 可按调用参数写入指定输出目录。

这些能力仍受 ToolRunner 的 JSON 配置开关和超时控制。

## 工具结果与前端

工具返回值自动进入下一轮 LLM 上下文。工具日志记录到用户历史日志目录。结果中的 `artifacts[]` 供前端生成文件、图片、音频或视频卡片；无 artifact 的结果仍可在工具调用卡片中查看。
