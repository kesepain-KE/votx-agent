# 智能体工作规范

更新时间：2026-07-12

本文是 `AGENTS.md` 的简明知识库说明。发生冲突时，以当前源码、两层 JSON 配置和根目录 `AGENTS.md` 为准。

## 配置与工具选择

配置权威：

```text
users/<name>/config.json > config/config_core.json > 代码默认值
```

`.env` 只用于启动级参数和兼容兜底。旧文件/网络沙箱环境变量不再是有效配置面。

执行任务前先判断是否存在对应 Skill，并读取其 `SKILL.md`：

```text
专用 Skill/工具 > 基础文件或网络工具 > shell
```

shell 用于没有等价工具的命令执行、诊断、构建和环境检查，不因旧版“受控 shell”文案假设存在硬编码命令黑名单。

## 默认数据落点

```text
临时文件:   <project-root>/tmp
输出文件:   <project-root>/users/<name>/download
上传/附件:  <project-root>/users/<name>/history/file
用户知识库: <project-root>/users/<name>/knowledge
全局知识库: <project-root>/knowledge
```

目录用途用于分类和默认落点，不等于路径白名单。仍需避免覆盖、移动或删除任务无关数据；不可逆操作先确认范围。

## 当前插件

```text
audio_universal  auto_improve       download_anything
file             image_edit         image_generation
kb_retriever     network            qq_file
qq_send          shell              skill_creator
speech_generation speech_to_speech  task_plan
```

当前源码没有 旧版内置文档转换、PDF 与 DOCX 插件。处理二进制文档时应说明需要用户 Skill、外部工具或独立服务，不能虚构已删除工具。

常用基础能力：

| Skill | 能力 |
|---|---|
| `file` | 读写、范围读取、编辑、目录、搜索、复制、移动、删除文件 |
| `shell` | 命令、cwd/env、stdin、会话状态 |
| `network` | HTTP GET/POST、网页正文读取 |
| `download_anything` | 直链检查/下载、视频/音频下载、下载记录 |
| `tavily_search` | 搜索、提取、爬取、站点地图、深度研究 |

## 当前限制口径

已解除：

- 文件目录白名单沙箱。
- shell 硬编码危险命令黑名单和工作目录沙箱。
- network 的 `network_scope` 与内网/SSRF 地址拦截。
- download 输出目录白名单。

仍保留：

- ToolRunner 的 `enabled`、`deny`、禁用 Skill 和 `tool_timeout`。
- 路径解析、文件存在/类型检查、超时、取消、日志和非法文件名清理。
- 隐私、密钥保护、不可逆操作确认和无关数据保护。

## 知识库维护

- 用户资料默认写入 `users/<name>/knowledge/`。
- 全局共享说明写入 `knowledge/`。
- 任一知识库增删改移后更新最近的 `data_structure.md`。
- 索引记录用途、路径和关键词，不复制正文。

## 最小验证

- Python：`python -m py_compile <changed.py>`
- 前端：`npm run build`
- 文档：检查路径、配置字段、插件名和源码一致性
- 同一命令连续失败 3 次时停止并报告
